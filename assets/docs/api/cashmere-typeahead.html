Additional classes

TypeaheadComponent extends HcFormControlComponent

Properties

Name Description

blur: EventEmitter<any>

Event emitted when the input box of the typeahead loses focus

debounceTime: number

The amount of time in milliseconds to delay between keystrokes before emitting the valueChange event for the input, default 200

emptyOptionSelected: EventEmitter<any>

Event emitted when the user hits enter and there is not an option selected (or no results available yet)

isDisabled: boolean

Enables or disables the component, default false

minChars: number

Number of characters required before the typehead will begin searching, default 1

optionSelected: EventEmitter<any>

Event emitted when an option is selected from the list of typeahead results

placeholder: string

Placeholder text for the input box of the typeahead, no default

required: boolean

Sets whether this is a required form element

showSpinner: boolean

Toggle to show and hide the searching spinner to provide the user with feedback (it does not automatically turn on and off, you need to manually control it), default false

silentFocus: boolean

Whether the focus event should emit a valueChange event and show the results. If set to true, then it will not emit the event nor show results until the user modifies the value in the input or the input value is empty. This is necessary to enable a user to be able to tab through the typeahead without clearing its value. Default is true.

value: string

Get or set the value of the select component

valueChange: EventEmitter<any>

Event emitted after each key stroke in the typeahead box (after minChars requirement has been met)
